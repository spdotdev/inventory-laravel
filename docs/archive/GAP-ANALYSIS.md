## Tier 1 — correctness / security / stability
- [x] **T1 (Laravel) Reset-token expiry is silently disabled under Carbon 3.** `ResetPasswordController.php:~40` uses `now()->diffInMinutes($record->created_at) > 60`; Carbon 3 `diffInMinutes` is signed, so a past `created_at` yields a negative value and the `> 60` TTL check never fires — expired reset links work indefinitely. Fix with an explicit comparison (`Carbon::parse($created_at)->lt(now()->subMinutes(TTL))` or `diffInMinutes(absolute: true)`). Add tests: valid / expired / tampered token, and "existing Sanctum tokens revoked after reset".
- [x] **T2 (Laravel) `/errors` is unauthenticated, unthrottled, unbounded.** `routes/api.php` (`POST /errors`) sits outside auth + throttle; unbounded inserts into `inventory_client_errors`. Add a dedicated `throttle:inventory-errors` limiter (key device_id+IP), a scheduled prune command (retain N days), and tests (422 on bad payload, throttle fires, prune deletes old rows).
- [x] **T3 (Laravel) Non-atomic `remove`/`move` stock mutation.** `ProductController::remove` does read-modify-write (`quantity = max(0, q-amt); save()`) — concurrent removes drop a decrement. `add` is already atomic. Make remove atomic (`update(['quantity' => DB::raw('GREATEST(quantity - N, 0)')])` then refresh); check `move` re-save too. Test the flooring + result shape still hold.
- [x] **T4 (Android) 401 mid-session strands the user.** `AuthInterceptor` clears the token on 401 but `AuthViewModel.authenticated` is read once at init and nothing observes the token reactively — expired session = every call silently 401s on authed screens until cold restart. Make `TokenStore` expose a `StateFlow` (or emit an unauthorized event) that `AuthViewModel` collects to flip `authenticated=false`; existing `MainActivity` redirect handles the rest. Add a MockWebServer test: 401 → token cleared → back to auth.
- [x] **T5 (Android) No network-down UX; raw errors leak.** VMs surface `error.message` (e.g. "Unable to resolve host"). Add a shared error mapper (IOException → friendly "can't reach server", HttpException by code) reused across VMs, plus a Retry affordance on read screens. Unit-test the mapper.

## Tier 3 — UI / UX / design / i18n
- [x] **T9 (Android) Pull-to-refresh spinner fires on every mutation.** Split a dedicated `refreshing` flag from generic `loading` across all 7 refreshable screens (Storage/Location/Products/Dashboard/Missing/AllStorages/Households). `HierarchyStore.refresh(userInitiated)` flips `refreshing` only for user pulls (post-mutation reloads stay silent); the `launchLoading` VMs and ProductDetail.load() gained the same distinction. Screens bind `isRefreshing = state.refreshing`; `loading` still drives inline progress + empty-state gating. Gated-repo unit test asserts create()→loading-not-refreshing, refresh()→both. CI verifies compile/tests.
- [x] **T10 (Android) Transient errors are sticky/silent + not announced.** a11y: the shared `ErrorRetry` (used across ~9 read screens) and the two auth screens' inline error `Text` now carry `semantics { liveRegion = Assertive }`, so TalkBack announces failures the moment they appear. One-shot action errors: added a reusable `SnackbarErrorEffect` (+`SnackbarHost`) component and wired it into `ProductDetail` — the canonical action-error surface (save/upload/delete, which have no sensible inline retry); a `consumeError()` clears it after showing so it can't re-fire. Design split, recorded: load-failure screens keep the inline `ErrorRetry` (retry affordance a transient snackbar can't offer, now announced); pure-action results use the snackbar. Material3 Snackbar is itself a live region, so no extra semantics needed there. Unit test covers `consumeError()`.
- [x] **T11 (Android) Storage types shown raw + unlocalized.** Added `storage_type_{freezer,fridge,pantry,other}` strings to `values/` **and** `values-nl/` (Vriezer/Koelkast/Voorraadkast/Overig), plus a reusable `storageTypeLabel(type)` composable (backed by a unit-testable `storageTypeLabelRes`) that maps the transport enum → localized label with a capitalized-raw fallback for unknown/future values. Applied at all 3 render sites (StorageOverview label + add-sheet FilterChip, AllStorages label); the raw enum stays on the wire. Unit test guards the mapping and that it stays in lockstep with `STORAGE_TYPES`.
- [x] **T12 (Android) Complete the Dutch translation.** Added the 3 missing NL strings — `dashboard_favorite_shelves` (Favoriete planken), `drawer_no_households_hint`, `products_pane_swipe_hint` (Veeg een product naar links…). Verified locales are now in exact lockstep (140/140 keys, no locale-only keys) with matching `%`-format-arg counts on every key. (Android lint's `MissingTranslation` is the standing CI guard against future drift.)
- [x] **T13 (Android) Config-change/process-death polish.** Converted the dialog/sheet visibility flags to `rememberSaveable` (StorageOverview add-sheet, LocationDetail add-shelf/add-product sheets, Households create-sheet, ProductDetail delete-confirm, Settings sign-out confirm) so they survive rotation; left the transient clipboard-`copied` toast and the `SortMenu` dropdown as bare `remember` (persisting those would be wrong). Fixed the `MainActivity` auth redirect: it now fires only on a real login/logout **transition** via a pure, unit-tested `authRedirectFor(previous, current)` + a `rememberSaveable` last-auth marker — on process-death restore `previous == current` so it no-ops and the NavController's restored back stack survives (no more bounce to the dashboard). `AuthRedirectTest` covers cold-start/login/logout/restore/no-change.

## Tier 4 — CI / CD / release stability
- [x] **T14 (Laravel) Health check has no dependency probe.** `HealthController` now runs a `SELECT 1` and returns `{name, api, status, database}` — DB reachable → 200 `status: ok`; DB down → **503** `status: error, database: unavailable` (raw error logged via `report()`, never leaked). Redis intentionally not probed (the package doesn't own the host's cache/queue config). `HealthCheckTest` mocks the `DB` facade so **both** paths run without a real driver (ran green locally, not just CI). Contract updated.
- [x] **T15 (CI/Laravel) Add a MySQL CI job.** Added a `mysql` job to `ci.yml` running a MySQL 8 service container (health-gated) that executes the **full** PHPUnit suite against real MySQL — so migrations actually run on MySQL and the ENUM storage-type column + `ON DELETE CASCADE` tree are exercised on the prod engine, not just SQLite. `TestCase` now honors `DB_CONNECTION=mysql` (+ `DB_*` env) to flip the driver; default stays in-memory SQLite for the fast `quality` job. Runs on the same push/PR triggers. Local gates (Pint/Larastan + default-path test) green; the MySQL run itself executes on CI.
- [x] **T16 (CI/Android) Instrumented flow tests never run in CI.** 27 `androidTest` flow tests exist but CI ran only `testDebugUnitTest`. Added `.github/workflows/instrumented.yml`: a KVM-accelerated emulator job (API 34, `reactivecircus/android-emulator-runner` with AVD caching) running `connectedDebugAndroidTest` on a **nightly** cron + `workflow_dispatch` (kept off per-PR CI since emulator boot is slow). Restores the debug keystore from the secret, guards on the canonical repo so forks skip, uploads the test report artifact always. Emulator run is CI-only (unrunnable locally); YAML validated. NOTE: the nightly may surface historically-flaky flow tests (e.g. SearchFlowTest nav) to triage — that's the job doing its job, tracked separately if it recurs.
- [x] **T17 (CI/Android) Release + fork-PR pipeline.** `ci.yml` now runs unit-tests + lint (the real gates, no secret needed) first, then gates the keystore-restore / `assembleDebug` / APK-upload steps on a `Check for signing secret` step — so a secret-less **fork PR** goes green instead of red-failing on signing infra it can't have. `release.yml` now (a) fails loudly with a clear message if the signing secret is absent (a real release genuinely needs it) and (b) publishes the debug-signed APK as a **prerelease** with a "⚠️ Preview build — debug-signed, not Play-ready" note, since real release signing + AAB is tracked in ROADMAP. Both YAMLs validated; CI verifies the runs. (`debug-apk.yml` is manual-only in the canonical repo — left as-is.)

## Tier 7 — found during CI verification (branch pushed → PRs opened)
- [x] **(laravel) Unsigned-underflow in atomic `remove()`** — the **T15 MySQL job caught a real bug on its first run**: `CASE WHEN quantity - N < 0` underflows on `BIGINT UNSIGNED` (MySQL strict mode, SQLSTATE 22003) when N > quantity; SQLite silently passed. Fixed to compare before subtracting (`quantity < N`). Vindicates T15.
- [x] **(android) Missing `mutableStateOf` import in `MainActivity`** — the T13 auth-gate used `mutableStateOf` without importing it; CI compile caught it (Gradle unrunnable locally). Import added.
- [x] **T22 (CI/android) `dependency-review` red-fails every PR on infra.** `dependency-review.yml` hard-errors "not supported on this repository" because the Dependency Graph (GHAS on private repos) isn't enabled — infra, not code, red-failing every PR (same class as T17's fork-PR signing red). Made the job `continue-on-error` so it doesn't block; once a maintainer enables the Dependency Graph in Settings → Security, real high-severity findings surface. (Enabling the setting itself is a maintainer action — out of scope for this branch.)
