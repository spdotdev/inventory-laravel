Markers: `[ ]` todo · `[x]` done · `[~]` deferred (record why). IDs are `W#` (wave-2).

## Tier A — functionality actually broken against the real backend
- [x] **W1 (laravel+docs) Search tap-to-open is dead in production.** `SearchResultResource` returns only `id,name,quantity,location,shelf,path` — **not** `household_id`/`location_id`/`shelf_id`. The Android `SearchScreen` gates the clickable card on `household_id != null && shelf_id != null`, so against the real server those are always null → **every search result is non-clickable**; the headline "tap a hit to jump to the product" flow silently does nothing. It looks green only because the instrumented fixture `search_results.json` hand-injects those IDs (this is the "flaky SearchFlowTest nav" T16 waved off — it's a real contract bug, not flakiness). Fix: add `household_id`/`location_id`/`shelf_id` to `SearchResultResource::toArray` (additive, backward-compatible); reconcile `api-contract.md`; add a server test asserting the IDs are present so the mock can't mask it again.
- [x] **W2 (laravel) Invite link points at a `/join/{code}` route that doesn't exist.** `HouseholdController::invite()` advertises `https://{domain}/join/{code}`, but `routes/web.php` only registers `/` and `/reset-password` → any recipient opening the link in a browser gets a hard 404 at the exact moment the product needs a graceful "get the app / here's the code" fallback. Fix: add `Route::get('/join/{code}')` rendering a small Frost-styled Blade page (shows the code, links to the app); wire the route name. Test the page renders with the code.
- [x] **W3 (android) Home/AllStorages renders every load failure as an empty state.** `DrawerViewModel` maps `HierarchyStore.state` but drops the store's `error` (not even in `DrawerUiState`); `AllStoragesScreen` shows "No storages yet" on a network failure — indistinguishable from a real empty account, and pull-to-refresh failures vanish. Fix: add `error` to `DrawerUiState`, map it, render via the existing `ErrorRetry`, gate the empty text on `error == null`. Unit-test the mapping.
- [x] **W4 (android) Missing-items screen swallows load errors → false "all stocked".** `MissingItemsViewModel` exposes `error` but `MissingItemsScreen` never renders it; a failed load falls through to `missing_items_empty`. For a screen whose whole job is surfacing warnings, showing a false "all good" on error is the worst failure mode. Fix: render `state.error` via `ErrorRetry(onRetry = refresh)` above/instead of the empty state.
- [x] **W5 (laravel) Shelf `position` is never assigned on create → undefined tab order.** The client sends only `name`; `ShelfController::store` never computes `position`, so every shelf is created `position = 0`. `index` does `orderBy('position')` — all tied at 0 → the Shelves tab/pager order is undefined and can reshuffle. Fix: default `position` to `max(position)+1` within the location on create. Test that sequential creates get increasing positions.
- [x] **W6 (laravel) Households orphaned forever when the last member leaves.** `leave()` unconditionally detaches; when the last member goes, the household + its whole location→shelf→product tree survive with zero members, unreachable by anyone (tenancy 404s non-members) — dead data that only grows, inconsistent with the hard-delete posture. Fix: after detach, if `users()->count() === 0`, `delete()` the household (CASCADE cleans the tree). Test last-member-leave removes the tree.

## Tier B — error-surfacing + accessibility consistency (propagate the wave-1 patterns)
- [x] **W7 (android) Empty-state and error-state conflated on the two list screens that do show errors.** `StorageOverviewScreen` shows `ErrorRetry` *and* `storage_overview_empty` on a failed load ("Something went wrong / No storages yet"); `LocationDetailScreen` shows the shelf-load error *and* `location_no_shelves`. Fix: gate the empty text on `state.error == null` in both.
- [x] **W8 (android) Inline error text is invisible to TalkBack on six screens.** The wave-1 `liveRegion=Assertive` pattern wasn't propagated to the bare `Text(error)` on Dashboard, Households, Search, Invite, LocationDetail, Settings (join error *and* success). Fix: route list-screen errors through `ErrorRetry`, or add `Modifier.semantics { liveRegion = Assertive }`, matching the AuthScreen precedent; announce the Settings join-success too.
- [x] **W9 (android) Shelf tabs signal missing-stock by color + a 6dp dot only (WCAG 1.4.1, TalkBack-silent).** `LocationDetailScreen` marks an out-of-stock-mandatory shelf with red tab text + a decorative Box dot with no semantics. Fix: add a `stateDescription`/`contentDescription` on the Tab when it has a warning (e.g. "<shelf>, has missing items"), reusing the drawer's "⚠ <name>" pattern.
- [x] **W10 (android) Drawer/home location-delete failure is silently swallowed.** `DrawerViewModel.deleteLocation` has `runCatching{…}.onSuccess{ refresh() }` with no `onFailure` — a failed swipe-to-delete on AllStorages leaves the location with no feedback. Fix: surface the failure (feed the store `error` / one-shot event) consistently with W3.

## Tier C — backend correctness / hardening
- [x] **W11 (laravel) `LIKE` wildcards unescaped in product + user search.** `SearchController` and `AdminController` interpolate the raw term into `%…%`; a user-supplied `%`/`_` acts as a wildcard (searching `50%` over-matches; a lone `%` returns everything). Bound, so no injection — but wrong results. Fix: `addcslashes($term, '%_\\')` before interpolating at both call sites; test `%`/`_` are treated literally.
- [x] **W12 (laravel) Login is a timing-based user-enumeration oracle.** On a missing email, `login()` throws before any `Hash::check`, so non-existent accounts respond measurably faster than wrong-password ones — inconsistent with the non-enumerable `/forgot-password` + 404-everywhere posture. Fix: run a dummy `Hash::check` against a constant hash when the user is null so both paths do equal work before the same `auth.failed`.
- [x] **W13 (laravel) Email normalized inconsistently across auth flows.** Register/login store+lookup verbatim; forgot-password lowercases; Google matches verbatim. Masked on MySQL (CI collation) but broken on the SQLite the package is CI-tested on: register `Foo@x.com` then reset/login with other casing silently misses. Fix: normalize to lowercase once at the boundary (`prepareForValidation()` in Register/Login requests + lowercase the Google claim). Test mixed-case register→login→reset.
- [x] **W14 (laravel) Unbounded stock `add` can overflow `UNSIGNED INT` → 500.** `amount` is `min:1` with no `max`; `quantity` is `unsignedInteger` (~4.29B); a large/repeated add throws MySQL "out of range". Fix: add a sane `max` to `amount` (and `quantity` in `ProductRequest`). Test the cap rejects over-max.

## Tier D — test coverage of the above edge paths
- [x] **W15 (laravel) Backend edge-path tests + orphaned-image decision.** Add: stock `amount` 0/negative/missing rejected on add+remove; Google-only account rejected on password login; (with W13) email case-insensitivity. Also decide the orphaned-image-on-`destroy` case: product/shelf/location delete cascades the row but never calls `deleteStoredImage`, orphaning the file — either wire best-effort cleanup into `destroy` or explicitly document the orphan. Implement the decision + test.
- [x] **W16 (android) Error-path unit tests (pure-JVM, no emulator).** No error-path tests exist for `HierarchyStore`/`DrawerViewModel`/`MissingItemsViewModel`. Add: `HierarchyStore.refresh` sets `error` + clears `loading`/`refreshing` on throw; `DrawerUiState.error` reflects it (W3); `MissingItemsViewModel` exposes it (W4); `DrawerViewModel.deleteLocation` surfaces failure (W10).

## Tier E — contract accuracy + CI parity
- [x] **W17 (docs) `api-contract.md` omits live, client-required response shapes.** Despite the "Reconciled" banner it never documents `join_code` on the household resource (client `HouseholdDto.join_code` is *required* — a rename silently breaks deserialization), `position`+`location_id` on shelves, `avatar_url` on the user, or the search nav IDs (W1). Fix: add explicit response-shape blocks for Household, Shelf, User, and the reconciled Search result.
- [x] **W18 (CI/both) No `concurrency:` cancellation on any of the 9 workflows.** Rapid PR pushes stack redundant runs instead of superseding — wasteful, and expensive for the KVM emulator + MySQL-service jobs. Fix: add `concurrency: { group: <workflow>-<ref>, cancel-in-progress: true }` to each (keep `false` on `release.yml`).
- [~] **W19 (CI/android) Kotlin style/static-analysis ungated (asymmetric with Laravel's Pint+Larastan).** Android CI runs only `testDebugUnitTest` + non-fatal `lint`; no ktlint/detekt. Fix: add ktlint + detekt Gradle tasks and gate them, matching the Laravel quality job. **DEFERRED (wave-2):** gating requires a baseline generated against the real codebase, and this environment has no working Android toolchain (JDK 26 vs the project's Gradle → `./gradlew` fails to start), so I can neither generate the baseline nor verify the result. Adding a detekt/ktlint plugin to `build.gradle.kts` blind would affect plugin resolution for *every* Gradle invocation (including the compile/test job) and gate on an unknown violation count — either of which could redden the whole Android pipeline with no way for me to confirm green before the loop's completion promise. Deferred rather than gamble an unverifiable red. Follow-up: on a machine with a compatible JDK, run `detektBaseline`/`ktlintCheck`, commit the baseline, then flip the gate on. Tracked in `inventory-android/ROADMAP.md`.
- [x] **W20 (CI/laravel) CI trigger filters skip workflow-only changes.** `ci.yml` `paths:` omits `.github/workflows/**`, so a PR editing only a workflow doesn't run the suite → CI changes land unvalidated. Fix: add `.github/workflows/**` to the `pull_request` paths.
- [~] **W21 (CI/both) No coverage measurement or reporting.** Both PHP jobs set `coverage: none`; Android has no jacoco. Silent erosion of the load-bearing auth/tenancy/stock-action coverage is invisible. Fix: enable PCOV/`--coverage-text` on the Laravel quality job + a jacoco report on Android (non-blocking report first). **PARTIAL (wave-2):** PHP coverage done (PCOV + `phpunit --coverage-text` on the quality job, `<source>` include added). **Android jacoco DEFERRED** for the same reason as W19 — it needs `build.gradle.kts` Gradle plugin changes and this environment has no working Android toolchain to verify them; blind changes could redden the pipeline unverifiably. Follow-up tracked with W19 in `inventory-android/ROADMAP.md`.

## Tier F — performance / UX polish
- [x] **W22 (android) Invite QR generated on the composition thread.** `InviteScreen` runs a 512×512 ~262k-pixel `setPixel` loop inside `remember(link)` on the composition thread → a visible hitch when the invite loads. Fix: `produceState(null, link){ value = withContext(Dispatchers.Default){ qrBitmap(link) } }` with a spinner until ready.
- [x] **W23 (android) Drawer "Search" silently searches only the first household.** `AppDrawer` wires Search to `entries.firstOrNull()?.id`; a multi-household user gets first-only results with no indication. Search is legitimately household-scoped, so this is a UX trap, not drift. Fix (no new endpoints): show the searched household on `SearchScreen`, or gate/replace the drawer entry when `entries.size > 1`.
